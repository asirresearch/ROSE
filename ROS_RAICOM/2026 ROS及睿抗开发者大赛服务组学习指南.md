## 3. 图像识别

图像识别是比赛中得分的关键环节。整个流程分为三个部分：**制作数据集 ➡ 训练模型 ➡ 部署到虚拟机**。

### 3.1 制作数据集 (Roboflow 平台)

推荐使用官方合作伙伴 **Roboflow** 网页端平台来进行数据集的制作和管理。

#### 1. Roboflow 的核心优势
*   **无需安装**：全网页端操作，支持团队多人协作，数据云端保存。
*   **AI 辅助标注（Label Assist / Smart Polygon）**：集成 Segment Anything (SAM 3) 等前沿大模型。标注时只需点击物体（如灭火器），AI 会自动精准勾勒出轮廓，省去手动拉框的繁琐。
*   **批量自动标注（Auto Label）**：支持使用 Grounding DINO 等零样本模型。只需输入英文提示词（如 `fire extinguisher`），系统即可自动在数百张图片中检测并生成初始标注框，人工只需进行少量审核微调。
*   **一键数据划分**：自动将数据按比例（如 70% 训练、20% 验证、10% 测试）划分，并自动生成 YOLOv8 训练专用的 `data.yaml` 配置文件。

#### 2. 数据标注与导出步骤
1.  **创建项目**：在 Roboflow 注册并创建新项目，项目类型选择 **Object Detection（目标检测）**。
2.  **上传图片**：将拍摄或收集的原始图片拖拽上传。如果有已标注好的其他格式数据，可将 `images` 和 `labels` 文件夹同时拖入，Roboflow 会自动读取并映射类别。
3.  **进行标注**：使用快捷键 `W` 绘制边界框，或者使用 `Smart Polygon`（魔棒图标）进行半自动标注。
4.  **生成并导出数据集**：
    *   标注完成后，点击左侧菜单的 **Dataset -> Generate** 生成一个新版本。
    *   生成后，点击右上角的 **Export Dataset**，在 Format 菜单中选择 **YOLOv8**。
    *   选择 **download zip to computer** 下载到 Windows 本地，或选择 **show download code** 直接在虚拟机中通过 Python 命令下载。

### 3.2 训练模型 (Windows GPU 宿主机)

为了利用本地显卡（如 RTX 显卡）提高训练速度，建议在 Windows 宿主机上进行模型训练。

#### 0. yolov8教程（选学）
【yolov8目标检测原理与实战（训练自己的数据集）】 https://www.bilibili.com/video/BV19nrbYjEyN/?share_source=copy_web&vd_source=a59b613664a3ad7a3cd1678c2c891bda

#### 1. 安装 Anaconda（或 Miniconda）
*   前往 [Anaconda 官网](https://www.anaconda.com/) 下载并安装。
*   在安装选项中，注意勾选“将 Anaconda 添加到环境变量”，以便在终端直接使用 `conda` 命令。

#### 2. 创建 Conda 虚拟环境
打开 Anaconda Prompt，运行以下命令创建专门的训练环境：
```bash
# 创建名为 yolov8 的虚拟环境，指定 Python 版本为 3.8 或 3.9
conda create -n yolov8 python=3.8 -y

# 激活虚拟环境
conda activate yolov8
```

#### 3. 安装依赖与 YOLOv8
在激活的环境下，安装对应显卡 CUDA 版本的 PyTorch 以及 Ultralytics 库：
```bash
# 示例：安装适配显卡的 CUDA 11.8 版 PyTorch（请根据个人显卡配置调整）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118

# 安装 YOLOv8 官方框架
pip install ultralytics
```

#### 4. 准备数据集与修改配置
将 Roboflow 导出的数据集解压到项目目录下（例如命名为 `Ruicom`）。
打开 `Ruicom/data.yaml` 文件，将路径修改为**绝对路径**或使用 `path` 参数，避免 YOLO 找不到图片：
```yaml
path: D:/yolov8/Ruicom    # 数据集根目录
train: train/images       # 训练集路径（相对于path）
val: valid/images         # 验证集路径（相对于path）
test: test/images

nc: 2
names: ['fire', 'fire extinguisher']
```

#### 5. 编写训练脚本
在项目根目录下新建 `train.py`：
```python
from ultralytics import YOLO

def main():
    # 本赛项部署于 CPU 虚拟机，为了保证推理速度，必须使用 n (Nano) 系列权重
    model = YOLO("weights/yolov8n.pt") 

    # 开始训练
    results = model.train(
        data="Ruicom/data.yaml",    # 数据集配置文件路径
        epochs=100,                 # 训练轮数
        imgsz=640,                  # 输入图片大小
        batch=16,                   # 批次大小
        device=0,                   # 使用 GPU (0号卡) 训练
        workers=0,                  # 在 Windows 上设为 0 可有效防止“页面文件太小”报错
    )

if __name__ == '__main__':
    main()
```
运行 `python train.py`。训练完成后，最优权重保存在 `runs/detect/xxx/weights/best.pt`。

### 3.3 部署到虚拟机 (Ubuntu 20.04)

为避免依赖库冲突导致 ROS（如 `cv_bridge`）瘫痪，必须在虚拟机的隔离环境中运行 YOLO 识别服务。

#### 1. 搭建虚拟环境（无需安装臃肿的 Jupyter）
在 Ubuntu 终端中执行：
```bash
# 1. 安装 venv 支持工具
sudo apt update && sudo apt install python3-venv -y

# 2. 创建并激活虚拟环境
mkdir -p ~/reinovo_project
cd ~/reinovo_project
python3 -m venv yolo_env
source yolo_env/bin/activate
```

#### 2. 安装轻量级 CPU 推理依赖
在激活的虚拟环境 `(yolo_env)` 中，使用国内清华镜像源快速安装：
```bash
# 升级 pip
pip install --upgrade pip -i https://pypi.tuna.tsinghua.edu.cn/simple

# 安装 CPU 版 PyTorch (仅 ~190MB，无需下载数 GB 的 CUDA 驱动)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu

# 安装 YOLO 运行库、ROS 通信支持库以及 ONNX 运行环境
pip install ultralytics rospkg onnxruntime opencv-python -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### 3. 导入权重与性能提速
1.  **导出 ONNX**（推荐）：
    在 Windows 下将 `.pt` 模型转换，在 CPU 虚拟机上运行 ONNX 推理时间通常可以从 **4 秒大幅缩短至 120 毫秒左右**：
    ```python
    model = YOLO("best.pt")
    model.export(format="onnx") # 生成 best.onnx
    ```
2.  **传输文件**：将 `best.onnx`（或 `best.pt`）传入虚拟机的 `~/reinovo_project/weights/` 目录。

#### 4. 编写并运行 ROS 服务端脚本 (`yolo_service.py`)
在项目逻辑中，小车巡航到达目标点后，若实时高频检测会极大占用 CPU。**最合理的工程设计是使用 ROS 服务（std_srvs/Trigger）模式**：小车停稳后发送请求，YOLO 瞬间抓取一帧图片检测并返回结果（如 `"fire"`、`"fire extinguisher"`），检测完自动进入休眠，从而将全部算力归还给导航和 SLAM 算法。

将 `yolo_service.py` 放在 `~/reinovo_project` 下运行：
```bash
source ~/reinovo_project/yolo_env/bin/activate
python3 yolo_service.py
```

## 4. 附录 - Ubuntu 20.04 常用指令

本附录整理了开发和调试过程中最常用、最基础的指令，便于赛前快速查阅。

### 1. 文件与目录操作
*   `ls`：列出当前目录下的文件和文件夹。
    *   `ls -la`：列出详细信息（包括权限、大小、修改时间以及隐藏文件）。
*   `cd <目录名>`：切换工作目录。
    *   `cd ..`：返回上一级目录。
    *   `cd ~`：跳转至当前用户的家（Home）目录。
*   `pwd`：显示当前工作目录的绝对路径。
*   `mkdir <目录名>`：新建文件夹。
    *   `mkdir -p a/b/c`：递归创建多级文件夹。
*   `cp <源文件> <目标路径>`：复制文件。
    *   `cp -r <源文件夹> <目标路径>`：复制整个文件夹。
*   `mv <源路径> <目标路径>`：移动文件或重命名。
*   `rm <文件名>`：删除文件。
    *   `rm -rf <文件夹>`：强制并递归删除整个文件夹（**警告：请谨慎使用此命令**）。
*   `chmod +x <文件名>`：为文件赋予可执行权限（写好 ROS 脚本后必须运行此命令）。

### 2. 系统与环境管理
*   `sudo <指令>`：以系统管理员（root）权限执行该指令。
*   `sudo apt update`：更新本地软件包列表。
*   `sudo apt install <软件包名>`：在线安装系统软件包（如 `sudo apt install python3-pip`）。
*   `source <文件>`：使配置文件或环境变量立即生效（如激活虚拟环境：`source yolo_env/bin/activate`）。
*   `echo $ROS_PACKAGE_PATH`：打印环境变量，检查 ROS 工作空间是否配置成功。
*   `top` 或 `htop`：实时查看系统资源占用情况（CPU、内存、进程占用率）。

### 3. ROS 核心调试工具
*   `roscore`：启动 ROS 核心管理器（一切 ROS 通信的基石，必须最先启动）。
*   `roslaunch <包名> <launch文件名>`：运行 launch 配置文件，通常会同时启动多个 ROS 节点（如启动小车导航）。
*   `rostopic list`：列出当前活跃的所有话题（Topic）。
*   `rostopic echo <话题名>`：在终端实时打印指定话题的数据流（常用于调试传感器数据）。
*   `rosservice list`：列出当前活跃的所有服务（Service）。
*   `rosservice call /detect_object`：手动呼叫 YOLO 检测服务，用于测试识别效果。
*   `rqt_image_view`：ROS 图像查看器。可以通过下拉菜单选择查看相机原图或经由 YOLO 绘制边界框后的检测输出画面。
